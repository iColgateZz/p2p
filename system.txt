pub struct Transaction {
    pub hash: String,     // hash(data + timestamp)
    pub data: String,     // e.g. "Alice -> Bob: 5"
    pub timestamp: u64,   // when transaction was created
}

pub struct Block {
    pub hash: String,                       // hash(prev_hash + transactions + timestamp)
    pub prev_hash: String,                  // points to the previous block in the chain
    pub transactions: Vec<Transaction>,     // t that were pending at the moment of creating the block
    pub timestamp: u64,                     // when block was created
}

Block and transaction communication

    Scenario: Node creates a new transaction
        1. Set timestamp
        2. Generate hash
        3. Add to pending transactions
        4. Broadcast transaction to peers


    Scenario: Node receives a transaction T
        1. T exists?
        2.yes: Do nothing
        2.no:
            3. Add to pending transactions
            4. Broadcast transaction to peers


Scenario: Every X seconds, a node
    1. Generate block with pending transactions
    2. Add to blockchain
    3. Broadcast block to peers


    Scenario: Node receives a block B
        1. B exists?
        2.yes: Do nothing
        2.no:
            3. If block's hash != last block's hash, disregard the received block
            3. Add block to blockchain
            4. Remove included transaction from pending
            5. Broadcast block to peers


Scenario: Node joins the network
    1. Ask for peers
    2. Ask for hashes
    3. Ask for missing block to rebuild blockchain
    4. Start functioning as a normal node


In the background:
    1. Peer discovery  -> ask peers for their peers to expand the network
    2. Block discovery -> just in case the block did not reach the node
